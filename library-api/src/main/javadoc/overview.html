Pathling Library API

The Pathling Library API provides a comprehensive interface for performing analytics on FHIR healthcare data using Apache Spark. This library enables users to efficiently query, transform, and analyze FHIR resources at scale.

Features

  • Data sources - Read FHIR data from various formats including NDJSON, Parquet, Delta Lake, databases, and FHIR bulk export
  • Data sinks - Write processed data to multiple formats and destinations
  • SQL on FHIR - Create tabular views from FHIR data
  • Terminology functions - Integrate with FHIR terminology servers for code validation and expansion

Getting Started

The primary entry point for the library is the {@link au.csiro.pathling.library.PathlingContext} class, which provides access to all major functionality:


// Create a PathlingContext
PathlingContext pc = PathlingContext.create(spark);

// Read FHIR data
NdjsonSource patients = pc.read().ndjson("path/to/Patient.ndjson");

// Execute queries
FhirView view = FhirView.ofResource(ResourceType.OBSERVATION)
        .select(
                columns(
                        column("patient_id", "getResourceKey()")
                ),
                forEach("code.coding",
                        column("code_system", "system"),
                        column("code_code", "code"),
                        column("code_display", "display")
                )
        ).build();

Dataset<Row> result = data.view(view).execute();

Key Packages

  • au.csiro.pathling.library - Core API classes including PathlingContext
  • au.csiro.pathling.library.io.source - Data source implementations
  • au.csiro.pathling.library.io.sink - Data sink implementations
  • au.csiro.pathling.library.query - Query builder
